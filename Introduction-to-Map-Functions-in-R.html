Part I: Atomic Vectors

Definition: Atomic vectors (e.g. logical, integer, double, character) are homogeneous and follow a hierarchy.

Homogeneous

We can create simple atomic vectors with c()

int_vec <- c(1:4)
class(int_vec)
## [1] "integer"
char_vec <- c("a", "b", "c", "d")
class(char_vec)
## [1] "character"
  • But what if we include two different classes in one atomic vector?

Homogeneous

Mixing types of data yield coercive results.

c("a", 1, "b", 2)
## [1] "a" "1" "b" "2"
class(c("a", 1, "b", 2))
## [1] "character"
  • The vector is stored as a character vector
  • … but why?

Atomic Vector Hierarchy

The second feature of atomic vectors are their hierarchical structure, visually represented below:

Atomic Vector Hierarchy

Let’s create a logical atomic vector. We can do this by using c()

(logi_vec <- c(T, F, F, T))
## [1]  TRUE FALSE FALSE  TRUE

Atomic Vector Hierarchy

Now, let’s take the average. We see that it actually returns a value, instead of an error. This is because it has transformed our logical vector into an integer before computing the mean.

mean(logi_vec)
## [1] 0.5

as.integer(logi_vec)
## [1] 1 0 0 1

Indexing Atomic Vectors

Indexing is manipulating specific elements or atoms, either for reading or writing

  • Takes the format of vector[index]

Lists Structure

Indexing Lists

  1. Single bracket []
  2. Double bracket [[]]
  3. Named element $

Single bracket

Extracts a sub-list

str(list_practice[1:2])

Single bracket

Extracts a sub-list

str(list_practice[1:2])
## List of 2
##  $ a: int [1:3] 1 2 3
##  $ b: chr "a string"
  • Indexing a list with a single bracket will always return another list

Double bracket

Extracts component from list

str(list_practice[[1]])

Double bracket

Extracts component from list

str(list_practice[[1]])
##  int [1:3] 1 2 3
  • It removes a level of hierarchy

Named element

It works similarly to [[]], but you must use the name of the element you are interested in (without quotation marks)

list_practice$b
## [1] "a string"
list_practice[["b"]]
## [1] "a string"

map() functions

  1. Always returns a list
  2. First argument is a list or atomic vector
  3. Second argument is a function, formula, or vector
  4. Multiple types of map functions
